# IR + AI V12 — Module result trim spec

## Why this exists

The module results are informative, but they still read too much like a structured evidence export.
The goal of V12 is to preserve the intelligence while making the result easier to grasp quickly.

## 1. Module entry pages

### Keep
- module title
- one short paragraph on what the module tests
- one compact Foundation linkage block
- mode choice

### Collapse or demote
- visible lanes
- perspective coverage wall
- long answer instructions
- "What it does not claim" if it reads like a separate strange side box

### Preferred structure
1. title + one-paragraph explainer
2. compact Foundation linkage block
3. mode cards
4. collapsed "How to read these cases"
5. first question CTA / first question itself

## 2. Module result pages

### Replace this pattern
- long lane summary
- linkage block
- explanation / decision / actor-lens explanation
- what stays unresolved
- long selected framings dump

### With this clearer middle stack
1. top-line read
2. 3 lane reads with visible bars
3. one compact linkage summary
4. **Decisive calls** section
5. collapsed **Full evidence log**
6. retake / next-step CTAs

## 3. Decisive calls structure

Show only the most informative 4 to 6 calls.
Each item should include:
- case title
- lane
- card type (Explanation / Decision / Actor lens)
- chosen framing
- one sentence on what that choice implies

If there are more than 6 useful items, push the rest into the collapsed evidence log.

## 4. “What it does not claim” formatting rule

Do not leave it as an isolated odd box.
Use either:
- one integrated "Scope and limits" disclosure block
- or a matched two-column notes block with "What it measures" and "What it does not claim"

## 5. Actor-lens explanation rule

Say it once, clearly:
- actor-lens cards model another actor's strategic read
- they do not overwrite the user's own issue read

Do not repeat that explanation in multiple blocks.
