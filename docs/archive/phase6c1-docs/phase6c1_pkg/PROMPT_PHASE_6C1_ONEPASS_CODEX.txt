## What to implement

### 1. Atlas card simplification
Refactor Atlas content so cards become brief executive summaries.

Target card structure:
- pattern name
- one plain-English summary line
- 2–3 short driver bullets or tags
- one short note about likely pressure/shift
- CTA to open detail page

Cards should be understandable to a smart lay reader in a few seconds.

### 2. Atlas detail pages
Add clickable detail routes:
- `/explore/atlas/[id]`

Each pattern detail page should include:
- hero summary
- one simple illustrative visual fingerprint
- what usually drives this pattern
- how it often looks in Security
- how it often looks in Technology
- where it is often confused with neighboring patterns
- pressure-test questions
- links to neighboring patterns

Do not overbuild. Keep the layout editorial and clean.

### 3. Copy hygiene pass
Tighten Atlas and visible Standard-mode summaries so they sound more natural and less AI-coded.

Use the copy guide.
Avoid phrases like:
- “X matters”
- “you think X matters”
- “you place real emphasis…”
- “nearest-fit shorthand” above the fold
- repetitive “conditional / selective / mixed” phrasing without concrete explanation

### 4. Light visual storytelling improvement
Add at least one meaningful visual improvement to Atlas/detail surfaces.
Examples:
- a small fingerprint strip
- concise tilt rows
- compact driver/shift visualization

Do not use radar charts.
Do not add decorative dashboard clutter.

### 5. Copy guardrail test
Add a small test or set of tests that scan the relevant Atlas / visible summary surfaces for clearly flagged phrases.
Keep this lightweight and practical.

## Constraints

- compare-with-friends

- rarity / percentile features
- crisis lab
- new modules
- big repo-wide wording sweep
- analytics or data collection
