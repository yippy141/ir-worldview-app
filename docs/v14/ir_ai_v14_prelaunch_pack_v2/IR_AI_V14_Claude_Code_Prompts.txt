PROMPT 1 — RESULT-CARD HERO SYSTEM
Read docs/v14/IR_AI_V14_Project_Manager_Brief.md and inspect the current IR result page (app/results/[payload]/page.tsx), AI result page (app/ai/results/[payload]/page.tsx), and Profile (components/profile/profile-report.tsx). Implement a reusable result-card hero pattern that makes the first screen after completion feel like a reward rather than a report.

Visual direction: the card should feel editorial, not gamified. Specifically:
- A contained panel with the tradition/archetype color as a left border or top accent bar (2-4px). Use the existing CSS variables: --t-realist, --t-institutionalist, --t-constructivist, --t-cpe for Foundation; pick a tasteful accent for AI results.
- The label (e.g. "Strategic Realist") in Georgia serif at roughly 1.4-1.6rem, letter-spacing tight.
- Modifier chips ("Hedger", "Conditional Solidarist") as small muted pill elements below the label. Not bright colored badges. Use the existing tradition-chip pattern but keep it understated.
- A one-sentence summary in system-ui body font at standard size with comfortable line height.
- A tension/surprise callout in a slightly inset box or with a subtle left-border accent. Visually distinct from the summary but not loud.
- Share and "What's next" actions at the bottom of the card, not competing with the main read.
- Background: --panel or very close. No gradients, no heavy shadows.
- The card must NOT look like a Bootstrap card with a centered heading and "Your result is..." preamble. It should look like a pull-quote panel in a serious publication. Think Financial Times or The Economist data callouts.

Surprising finding logic for Foundation results:
1. First, check getActiveTensions(dimensionScores). If one or more tensions fire, use the first one. These fire when two strong dimensions contradict (e.g. high institutions + high security competition). They are the most interesting signal.
2. If no tension fires, use getSubtraditionAffinity(familyKey, dimensionScores). This tells the user which strand within their tradition they land in (e.g. "defensive realism" within realism).
3. If subtradition returns the generic fallback and feels uninformative, fall back to neighborOverlapTexts[familyKey][runnerUpKey].
Keep the callout under two sentences. Example: "You scored as a Strategic Realist, but your high restraint pull is unusual for that profile. That combination points toward defensive realism — the argument that overextension provokes more danger than it prevents." No caveat sentence in the card. Caveats go lower.

Surprising finding logic for AI Governance results:
Use getActiveAiGovernanceTensions(axisScores). If a tension fires, use it. If none fire, use nearest-alternative framing from lib/ai-governance-results-v2.ts. Same constraint: under two sentences, no caveat.

Apply to Foundation result page and AI result page first. If Profile has 2+ saved layers, use the same card grammar for the top Profile payoff. Move existing dense sections (dimension bars, atlas match, readings, flip analysis, pressure tests) below the card. Do not delete important content unless it is redundant.

Do not change scoring, schemas, payloads, or routes. Run npm run lint, npm run test, and npm run build.
