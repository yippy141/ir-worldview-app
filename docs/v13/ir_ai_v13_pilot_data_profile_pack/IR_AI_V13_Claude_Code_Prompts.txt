## Prompt 1 — Profile payoff / anchored-spread implementation

Read `plans/V13_SOURCE_OF_TRUTH.md`, `plans/V13_AUDIT.md`, and any Claude Design handoff files already stored under `docs/design/`.

Implement a Profile payoff pass.

Requirements:
1. Above the fold, Profile should have one integrated headline, one short synthesis paragraph, one dominant anchored-spread-style visual, and no more than three chips/facts.
2. If the user has 2+ saved layers, lead with the integrated Atlas-style pattern; show Foundation as “closest modeled baseline” or supporting cue.
3. If the user only has Foundation, keep Foundation as the main result and show modules/AI as next steps.
4. Remove duplicated stable/shift/reasoning sections. Keep one concise “what stayed steady / what shifted / open tension” block.
5. Move evidence, completed overlays, appendix, and full module summaries below the fold and collapse where appropriate.
6. Do not introduce radar charts, decorative constellations, dashboards, or faux-precise charts.
7. Preserve existing scoring and payloads.
8. Ensure mobile around 390px stacks cleanly.

Run `npm run lint`, `npm run test`, and stop. Summarize files changed and any visual compromises.

3. Detail page content should include: short definition, what this perspective prioritizes, nearby archetype, core disagreement, result implications, “current debates to watch,” and readings/resources.

7. Do not add any new scored quiz logic.

Run `npm run lint`, `npm run test`, and stop.
